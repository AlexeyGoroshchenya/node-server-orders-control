эндпоинт http://localhost:5000/api/order/create
POST
ожидаются в body: 
    car: STRING (255),
    model: STRING (255),
    year: STRING (255),
    capacity: STRING (255),
    drive: STRING (255),
    type: STRING (255),
    description: STRING(1000)
	phone: STRING (255)(здесь телефон заказчика в международном формате. например +74955082210)
Если такой пользователь еще не зарегистрирован, происходит его регистрация.

page: number

получение заказа по id заказа
эндпоинт http://localhost:5000/api/order/getById
GET
ожидается действующий токен и в query пареметр id(number)

изменение заказа 
эндпоинт http://localhost:5000/api/order/
PUT
ожидается токен оператора, 
ожидаются в body: 
	orderId: number,
	 status: STRING,
    description: STRING(1000) по ТЗ предполагается что отображается последнее примечание. чтобы не затирать пустой строкой
     случайно предыдущее значение, предлагаю делать так: при вызове интерфейса  внесения изменений в заказ, 
     отрисовывать в textarea текст из предыдущей версии description, которая до этого получена с сервера.
     при сохранении изменений отправлять value из textarea. если оператор не трогал примечание, пусть тот же текст отправляется на пересохранение

пользователи:
регистрация нового пользователя.
http://localhost:5000/api/user/registration
POST
ожидается токен админа и в body:
name: string,
 phone: number, (в международном формате например 74955082210)
 password: string,
 role: string (значения ADMIN, OPERATOR или USER)

вход в систему
http://localhost:5000/api/user/login'
POST
ожидается в body:
 phone: number,
 password: string,

проверка авторизован ли пользователь 
http://localhost:5000/api/user/auth
GET
ожидает действующий токен, отдает новый токен. токен живет сутки

устанавливает временный пароль, который отправляется по смс
http://localhost:5000/api/user/send
POST
ожидается в body:
 phone: number
Если такой пользователь еще не зарегистрирован, происходит его регистрация.
!!!-------------------------------------------- 
!!! на период тестирования запрос возвращает пароль, чтобы не расходовать баланс смс-рассылки. перед деплоем на продакшен нужно это изменить 
!!!--------------------------------------------

смс рассылка:

вкл/выкл тестового режима смс рассылки на странице https://smsc.ru/edit/ вкладка API. Галочка: Режим тестирования (виртуальная отправка без оплаты) 

Имя можете создать здесь https://smsc.ru/senders/ , но его нужно регистрировать у операторов. На МТС и Мегафон отправка возможна только от платных имен, с абонентской платой 2 000 рублей в месяц у МТС и 2500 рублей в месяц у Мегафон или можете подключить общее имя по цене около 20 руб за смс

Как начать пользоваться сервисом можете прочитать здесь https://smsc.ru/faq/157/
